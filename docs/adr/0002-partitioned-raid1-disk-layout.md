# Partitioned RAID 1 with mdadm + ext4

Both 2TB Samsung SSDs are partitioned identically and mirrored with mdadm RAID 1: a small /boot partition (~512MB), a root partition (~30GB), and the remaining ~1.8TB for /data. Filesystem is ext4. dm-integrity was considered for block-level checksums but dropped due to limited community knowledge and debugging risk; ZFS was rejected due to DKMS kernel-update risk on an unattended node with 8 GB RAM. Either can be reconsidered on future hardware (16+ GB RAM, controlled update schedule). The alternative disk layout -- a USB SSD for the OS with internal drives as pure data -- was rejected because it adds a physical failure point, consumes a USB port needed for backups, and Ansible makes OS reinstalls trivial anyway (preseed + playbook).
